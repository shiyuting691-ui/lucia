# ROLE_DEFINITION.md

本文件是正式系统唯一角色定义。页面、Agent、任务、审批、建议输出必须使用以下角色口径。

## 标准角色

| 标准角色 | 定义 | 可生成任务 |
|---|---|---|
| 管理层 | 负责业务方向、资源投入、风险叫停、跨部门决策。 | 是 |
| 推广/市场 | 负责内容、渠道、投放、线索入口和推广素材。 | 是 |
| 销售/顾问/学管 | 同一个销售转化岗位。负责需求判断、产品匹配、客户沟通、报价推进、成交转化、复购和转介绍。 | 是 |
| 产品/后台 | 负责产品目录、价格口径、话术边界、资料维护、系统数据和流程规则。 | 是 |
| 交付/老师 | 负责具体交付、课程辅导、资料产出、质量反馈和交付风险反馈。 | 是 |

## 学管与顾问规则

- 学管和顾问是同一个销售转化岗位。
- 不要把学管拆成售后岗位。
- 不要把学管写成交付岗位。
- 不要把学管写成“销售成交后承接”的下游角色。
- 学管和顾问的区别只允许体现在来源渠道、客户类型和跟进分工，不允许体现在岗位性质上。

## 别名归一

| 输入 | 标准角色 |
|---|---|
| 市场部、推广部、市场/推广 | 推广/市场 |
| 销售、销售部、顾问、销售顾问、学管、学管部 | 销售/顾问/学管 |
| 产品部、后台、后台/产品 | 产品/后台 |
| 老师、教研、交付部、老师/教研 | 交付/老师 |
| 老板、管理、管理层 | 管理层 |

## 强制校验

- 任务没有标准角色，不允许创建。
- AI建议没有 `responsible_role`，不允许展示。
- 角色无法归一时，显示：`暂无真实数据，无法判断。`
