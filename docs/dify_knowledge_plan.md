## 一、Dify 的定位

Dify 在本系统中**只负责知识库管理**，不承载业务流程。
业务状态、内容审核、任务管理由 Python 后端 + 数据库负责。

Dify 提供的价值：
- 产品知识的语义检索
- 销售话术的智能匹配
- 内容生成时的知识增强（RAG）
- 客服/销售辅助查询

---

## 二、知识库分类结构（7个）

| 编号 | 知识库名称 | 用途 | 更新频率 |
|------|-----------|------|---------|
| 01 | 产品知识库 | 产品介绍、适用人群、定价、卖点 | 产品迭代时更新 |
| 02 | 销售话术库 | 销售流程话术、推荐脚本、跟进话术 | 每月优化一次 |
| 03 | 客户异议库 | 常见拒绝理由 + 标准应对方式 | 每月收集更新 |
| 04 | 营销案例库 | 成功成交案例、学员故事、成绩案例 | 持续积累 |
| 05 | 小红书风格库 | 爆款结构模板、标题公式、标签策略 | 每季度更新 |
| 06 | 风控表达库 | 禁用词清单、合规表达替换、平台规则 | 平台政策变动时更新 |
| 07 | 学校节点库 | 学校考试时间、热门专业、学情特点 | 每学期开始时更新 |

### 📁 05_小红书风格库

建议整理以下内容：
- 爆款标题公式（痛点型/利益型/悬念型）
- 开头钩子句式
- 正文结构模板
- 高互动标签清单（按学校/专业/时间节点分类）
- 禁止模仿的风格（不要太"广告味"）

### 📁 06_风控表达库

必须维护，避免违规：
- 小红书/微信禁用词列表
- 承诺类表达替换方案
  - ❌"保证过" → ✅"有专属退款协议"
  - ❌"100%通过" → ✅"历史学员通过率高"
  - ❌"最专业" → ✅"深耕UK/澳洲市场X年"
- 敏感话题（学历造假、考试泄题）的安全表达

### 📁 07_学校节点库

建议格式：`07_UCL_2026_考试节点.txt`

每个学校的文件包含：
- 学期时间表
- 主要考试节点
- 论文提交截止日期
- 热门专业和对应课程特点
- 已知难点科目

## 四、文件命名规范

格式：`{知识库编号}_{学校或产品}_{内容描述}_{版本}_{日期（可选）}.{扩展名}`

示例：
```
01_学年包_客户介绍_v3.html
02_DP论文_销售手册_v4.html
03_价格异议_应对话术_v1.txt
04_案例_UNSW_金融_期末成功_20260520.txt
05_小红书_标题公式100条_v1.txt
06_风控_禁用词清单_微信_20260601.txt
07_UCL_2026年_考试节点.txt
```

**版本号规范**：
- `v1`, `v2`... 表示大版本（内容大幅更新）
- 无版本号 = 草稿或一次性文档

## 五、上传优先顺序

按以下顺序上传，优先确保核心内容可用：

1. **第一批（立即上传）**：01_产品知识库 的所有文件
2. **第二批（本周内）**：02_销售话术库 的所有文件
3. **第三批（下周）**：03_客户异议库（先用简单TXT整理）
4. **第四批（本月内）**：07_学校节点库
5. **后续持续积累**：04_营销案例库、05_风格库、06_风控库

## 六、哪些资料不应该混在一起

❌ **不要混合**：
- 对外展示资料 vs 内部销售手册（客户可能截图）
- 价格信息（会变动）vs 产品介绍（相对稳定）
- 过期版本 vs 当前有效版本

✅ **正确做法**：
- 内部文档标注 `_内部版_`
- 过期版本加 `_archived_` 前缀或直接从 Dify 中禁用

---

## 七、版本管理规则

1. 每次更新重要文档，在 Dify 中**上传新版本**，不要直接覆盖
2. 在数据库 `knowledge_docs` 表里记录 `version` 字段
3. 旧版本在 Dify 中设为**禁用**状态（不删除，方便回溯）
4. 重要文档每季度检查一次是否需要更新

## 九、Dify API 批量上传命令

获取 Dify API Key 和 Dataset ID 后，使用以下命令：

```bash
# 上传单个文件
curl -X POST 'https://api.dify.ai/v1/datasets/{DATASET_ID}/document/create_by_file' \
  -H 'Authorization: Bearer {DIFY_API_KEY}' \
  -F 'data={"name":"文件名","indexing_technique":"high_quality","process_rule":{"mode":"automatic"}}' \
  -F 'file=@/Users/meilucia/Desktop/营销资料知识库/文件名.pdf;type=application/pdf'

# 批量上传脚本（在项目根目录运行）
python scripts/upload_to_dify.py --dataset-id {ID} --api-key {KEY}
```

## 十、后续计划：接入 RAG

知识库建好后，可以在 `PlanningAgent` 和 `ContentGenerationAgent` 中接入 RAG：

1. 生成内容前，先查询 Dify 知识库获取相关产品信息和销售话术
2. 把检索到的内容作为上下文传给 Claude
3. 生成质量更高、更贴合实际产品的内容

接入示例：
```python
# 在 PlanningAgent 中
def _fetch_product_context(self, product_id: str) -> str:
    resp = requests.post(
        f"https://api.dify.ai/v1/datasets/{DATASET_ID}/retrieve",
        headers={"Authorization": f"Bearer {DIFY_API_KEY}"},
        json={"query": f"{product_id} 产品介绍 卖点 话术", "top_k": 3}
    )
    return "\n".join([r["content"] for r in resp.json().get("records", [])])
```
