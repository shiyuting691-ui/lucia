# Token 优化规则 · TOKEN_OPTIMIZATION

> 本文档说明系统如何控制 Claude API 的 token 消耗，保证在有限额度下高效运行。

---

## 核心原则

**只有真正需要 AI 理解的操作才调用 Claude。其余所有操作用 Python/SQL 完成。**

---

## 一、哪些操作不调用 Claude

| 操作 | 处理方式 | 原因 |
|------|---------|------|
| 上传文件保存 | Python 直接写文件 | 不需要 AI 理解 |
| 扫描知识库 | Python 读文件系统元数据 | 只需文件名/路径/大小 |
| 导入订单 CSV | Python/pandas + SQLAlchemy | 结构化数据，直接写库 |
| 导入咨询 CSV | Python/pandas + SQLAlchemy | 结构化数据，直接写库 |
| 统计热门学校/产品 | SQL GROUP BY | 聚合运算，无需 AI |
| 查询订单趋势 | SQL 时间范围查询 | 数学统计，无需 AI |
| 页面数据展示 | 直接读数据库 | 展示层不调用 AI |

---

## 二、哪些操作调用 Claude（受控）

| 操作 | Claude 读取内容 | Token 控制策略 |
|------|---------------|--------------|
| 生成资料摘要 | 单文档前3000字 | 每次最多10个文档；max_tokens=600 |
| 更新市场信号 | 统计数字 + 简短描述 | 只传入聚合结果，不传原始CSV |
| 生成营销内容 | 摘要 + 关键参数 | 优先读 summary，不读全文 |
| 每日工作流 | 当日上下文摘要 | 限定 5 个信号、3 个动作 |

---

## 三、知识库读取策略

```
生成内容时的读取顺序：

1. 先查 knowledge_docs.summary（摘要，<300字）
2. 摘要不够时，读对应文件前 2000 字
3. 绝对不读取整个 knowledge_base/ 目录

示例：生成 Final精准押题 的小红书内容
  - 读取：Final精准押题_产品说明.summary（约200字）
  - 读取：押题异议_话术.summary（约150字）
  - 读取：押题产品风控规则.summary（约100字）
  - 总计约 450 字输入给 Claude（相当于 ~600 tokens）
  - 不读取全文（全文约 3000 字 = ~4000 tokens）
```

---

## 四、每日工作流 Token 控制

DailyWorkflow 的 prompt 结构：

```
系统提示（缓存）：约 500 tokens，每次只扣 0.1x
当日上下文：
  - 订单统计（数字）：~200 tokens
  - 热门学校/产品：~100 tokens
  - 市场信号（5条）：~500 tokens
  - 产品摘要（1-2条）：~400 tokens
合计输入：约 1700 tokens/次（不含缓存）
```

相比全文读取（每次 >8000 tokens）节省约 80%。

---

## 五、新产品推广 Token 控制

ProductLaunchAgent 每次生成 5 种内容，总 token 约：
- 输入：推广参数(~300) + 产品摘要(~300) + 风控摘要(~150) = ~750 tokens
- 输出：5 篇内容 × 约 300 tokens = ~1500 tokens
- 合计：约 2250 tokens/次推广

---

## 六、禁止的高 Token 操作

❌ `[所有产品文档全文] + prompt` — 禁止  
❌ `[完整 orders.csv 内容] + 分析` — 禁止  
❌ `[所有历史内容] + 请生成今日内容` — 禁止  
❌ 每次内容生成都重新扫描知识库全文 — 禁止  

---

## 七、额度估算参考

| 操作 | 约消耗 tokens | 频率 |
|------|-------------|------|
| 生成摘要（10个文档）| ~6000 tokens | 偶尔（文档更新后）|
| 更新市场信号 | ~2000 tokens | 每天1次 |
| 生成每日内容（3条）| ~5000 tokens | 每天1次 |
| 新产品推广（1次）| ~2500 tokens | 按需 |
| **每日总消耗（正常）** | **~9000 tokens** | **每天** |

使用 claude-sonnet-4-6 每百万 token 约 $3，每日约 $0.027（约 ¥0.2）。
