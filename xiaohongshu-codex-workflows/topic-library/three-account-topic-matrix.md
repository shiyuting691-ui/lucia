# 三账号选题矩阵

## 学生号

定位：上游流量入口，表达真实焦虑、吐槽、求助和日常。

适合选题：

- 课程作业看不懂要求
- DDL 临近但没有头绪
- AI 率反复降不下来
- 小组作业队友不配合
- 留学生活里的崩溃瞬间

标题方向：

- `墨大 GEOG10001 assignment2 真的无从下手`
- `dissertation 还有两周一字没动，我有点慌`
- `AI 率改了一整天还是高，真的会崩`

正文结构：

1. 直接进入场景
2. 描述卡住的具体点
3. 说自己的尝试和困惑
4. 开放式求助或更新进度

不要写：

- 服务介绍
- 机构口吻
- 标准答案
- 夸张承诺

## IP号

定位：有温度的专业承接，像学姐、老师或顾问在分享过程。

适合选题：

- 某课程作业要求怎么拆
- rubric 到底看什么
- AI 率风险怎么自查
- dissertation 前期怎么定题
- 老师反馈看不懂怎么办

标题方向：

- `写 dissertation 前我会先看这 3 个地方`
- `KCL 这类 essay，最容易卡在 rubric 对应`
- `AI 率高不一定是改词，先看这一步`

正文结构：

1. 说明自己见过的真实场景
2. 描述当时最容易误判的点
3. 给出判断过程
4. 留一个自然问题引导评论

不要写：

- `手把手教你`
- `保过`
- `私信我`
- 大段服务介绍

## 业务号

定位：占搜索位、建立专业可信度、承接中下游明确需求。

适合选题：

- 课程难点拆解
- 作业要求风险点
- 案例复盘
- 时间线规划
- 结果反馈脱敏展示

标题方向：

- `墨大 GEOG10001 assignment2 要先拆地图逻辑`
- `英国 dissertation 开题前最容易漏这一步`
- `AI 率 78% 的稿子，风险通常不只在措辞`

正文结构：

1. 点出具体问题
2. 拆关键判断
3. 说明处理顺序
4. 给出风险提醒和评论区开放问题

不要写：

- 硬广式承诺
- 过度包装结果
- 正文直接导流
- 与学生号完全相同的语气
